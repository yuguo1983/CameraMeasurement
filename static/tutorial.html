📏 3DME — 双目立体视觉测量系统

使用双目摄像头对物体进行三维测量，无需接触即可获取目标点的空间坐标和距离

一、系统概述

3DME 是一款基于双目立体视觉的实时测量工具。它使用一个输出 3840×1080 画面的 UVC 双目摄像头（左右各 1920×1080），通过图像匹配算法计算视差，再利用三角测量原理计算出被点击点的三维坐标（X, Y, Z），进而测量任意两点间的距离。

整个系统采用 B/S 架构：后端 Rust 服务驱动摄像头并处理计算，前端浏览器负责交互界面。启动 EXE 后自动打开浏览器，所有操作在网页中完成。

二、硬件与环境要求

项目要求
操作系统Windows 10/11（64位）
摄像头UVC 兼容的双目摄像头，输出 3840×1080（左右各 1920×1080）
依赖工具ffmpeg（用于从摄像头抓取画面）
浏览器Chrome / Edge / Firefox（推荐最新版）
💡 检查 ffmpeg 是否已安装： 打开 cmd 输入 ffmpeg -version，如果显示版本信息则已安装；未安装请到 ffmpeg.org 下载并加入 PATH。

三、目录与配置文件说明

文件/目录用途
3dme.exe主程序，双击启动
config.toml摄像头参数配置文件
calibration/存放标定用的左右图像对
output/存放标定结果 stereo_calibration.toml

config.toml 详解

[camera]
resolution = [3840, 1080]        # 摄像头分辨率（宽，高）
baseline_mm = 60.0               # 左右摄像头光心之间的距离 (mm)
focal_length_px = 1200.0         # 焦距（像素单位）
principal_point = [1920.0, 540.0]  # 主点坐标（图像中心）
exposure_time_ms = 20            # 曝光时间 (ms)
gain = 1.0                       # 增益

[camera.distortion]
k1 = 0.0  ; k2 = 0.0             # 径向畸变系数
p1 = 0.0  ; p2 = 0.0             # 切向畸变系数
k3 = 0.0

[stereo]
block_size = 9                   # 匹配块大小（越大越平滑，越慢）
max_disparity = 128              # 最大视差搜索范围
min_disparity = 0                # 最小视差

[reconstruct]
depth_min_mm = 100.0             # 最小测量深度 (mm)
depth_max_mm = 5000.0            # 最大测量深度 (mm)
⚠️ 关于 baseline 和 focal_length：
baseline_mm = 两个摄像头物理光心之间的距离，用尺子测量即可。
focal_length_px（焦距像素）和 principal_point（主点）需要知道相机内参。如果不确定，可以先用下表给出的典型值估算，再通过标定流程校准。
主点一般就是图像中心：1920×1080 的画面，主点 ≈ (960, 540)。

四、基本使用流程

1 启动程序
双击 3dme.exe，系统会自动打开浏览器进入主界面。如果未自动打开，手动访问 http://127.0.0.1:3030。
🏠 主界面示意图：左侧为控制面板，右侧为图像显示区
2 设置相机参数（如需）
在左侧面板中：
  • Camera Index — 如果你的电脑有多个摄像头，调整此编号选择双目摄像头（通常为 1）
  • Resolution — 选择 3840×1080（双目模式）或 1920×1080（单目）
3 拍摄画面
点击 按钮拍摄一帧立体画面。拍摄完成后左侧图像区会显示当前画面。
4 添加测量点
在图像上单击任意位置，系统会自动计算该点的三维坐标（X, Y, Z），并显示在左侧 Measurement Points 列表中。 重复点击可以添加多个测量点。
5 查看距离
添加 2 个或更多点后，Distances 面板会自动计算所有点对之间的三维距离。 每对点之间会在图像上以彩色虚线连接，每条线的颜色和虚线样式都不同，方便区分。
6 清空测量
点击 按钮可清除所有测量点和连线，开始新一轮测量。

其他功能

  • Live — 实时模式，连续拍摄并刷新画面（每秒约 60 帧），方便调整摄像头角度
  • Refresh — 刷新当前画面显示
  • Save — 将当前左右画面保存到 calibration/ 目录，用于标定
  • 视图切换 — 在图像上方可切换 Stereo View（左右并排）/ Left Only / Right Only / Disparity（视差图）

五、如何标定（Calibration）

标定的目的是获取准确的 焦距（focal_length_px） 和 主点（principal_point），提高测量精度。需要使用棋盘格标定板。

5.1 准备标定板

项目自带了一个棋盘格生成脚本，也可以直接打印一个棋盘格：

# 生成 9×6 棋盘格，每个格子 30mm
python gen_chessboard.py

将打印好的棋盘格贴在平整硬板上。

5.2 采集标定图像

1 将双目摄像头对准棋盘格，确保棋盘格完整可见
2 调整角度，使棋盘格在画面中占 1/3 ~ 1/2 面积
3 点击 Capture 拍摄一帧
4 点击 Save，图像将自动保存到 calibration/ 目录
5 改变棋盘格的位置/角度/距离，重复步骤 2~4，采集 至少 10 组 不同姿态的图像对
💡 拍摄技巧：
  • 让棋盘格出现在画面的不同位置（左上、右下、中间等）
  • 倾斜角度变化尽量大（0°~45°）
  • 距离摄像头 200mm ~ 1000mm 之间变化
  • 确保棋盘格表面光照均匀，不要有强反光

5.3 执行标定

1 在左侧面板的 Save Path 输入框中确认目录名为 calibration
2 点击 按钮
3 标定完成后，Stereo Parameters 面板会显示最新的 baseline、焦距和主点
4 标定结果自动保存到 output/stereo_calibration.toml

5.4 加载已有标定

如果之前标定过并保存了结果，启动程序后点击 Load Cal 即可加载已有的标定参数，无需重新标定。

六、测量原理（通俗版）

双目视觉模拟人眼的工作方式：

  1. 左右两个摄像头从不同角度拍摄同一场景
  2. 对于左图中的某个点，在右图中找到"同一个"点，两者在水平方向的像素差称为视差（disparity）
  3. 视差越大，物体离摄像头越近；视差越小，物体越远
  4. 利用三角测量公式：Z = baseline × focal_length / disparity，即可算出深度
  5. 有了深度 Z，再结合像点坐标，反算出 X 和 Y
📐 关键公式：
深度 Z = (基线距离 × 焦距) / 视差
X = (左图像点.x - 主点.x) × Z / 焦距
Y = (左图像点.y - 主点.y) × Z / 焦距

七、常见问题（FAQ）

❓ 启动后无法打开摄像头

  • 确认摄像头已正确连接 USB
  • 确认 ffmpeg 已安装且可在命令行使用
  • 尝试调整 Camera Index（如 0、1、2）
  • 检查 config.toml 中的分辨率是否与摄像头实际输出一致

❓ 添加测量点时提示 "Failed to add point"

  • 需要先点击 Capture 拍摄一帧画面
  • 需要先点击 Calibrate 或 Load Cal 加载标定参数
  • 如果点选在缺乏纹理的区域（如纯色墙壁），系统可能无法找到匹配点

❓ 测量数据不准确

  • 检查 config.toml 中的 baseline_mm 是否与实际物理基线一致
  • 进行标定获得准确的焦距和主点
  • 确保左右图像之间有足够的重叠区域
  • 尽量选择有明显纹理特征的点

❓ 标定失败

  • 确保 calibration/ 目录中至少有 3 组有效的左右图像对
  • 图像命名格式必须为 left_01.png + right_01.png（成对出现）
  • 棋盘格必须在图像中完整可见

❓ 程序闪退

  • 在命令行中运行 3dme.exe 查看错误日志
  • 检查 config.toml 格式是否正确
